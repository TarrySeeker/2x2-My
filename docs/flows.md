## 1. Flow: «Купить визитки» (pricing_mode = fixed)

**Персона:** Ирина, владелица кофейни в Ханты-Мансийске. Нужны визитки
для персонала, 1 000 штук. Уже знает, чего хочет.

### 🟢 Happy path

1. Ирина заходит на `/` (главную Yna), видит Hero
2. Кликает в header'е `[Каталог]` или в секции «Популярные услуги» (если добавим)
3. → `/catalog`
4. Видит `CategoryChips`, кликает `[Полиграфия · 8]`
5. → `/catalog?category=poligrafiya`
6. URL обновился, без перезагрузки (router.push с shallow)
7. Grid карточек обновился, показано 8 товаров полиграфии
8. Находит карточку «Визитки Стандарт», кликает
9. → `/product/vizitki-standart`
10. Видит галерею слева, инфо+калькулятор справа
11. В калькуляторе (тиражные ступени) выбирает `● 1 000 шт — 1 700 ₽`
12. ═ `POST /api/products/calculate` с `{ tiraj: "1000" }` (debounced 300 ms)
13. Итоговая цена обновилась: `Итого: 1 700 ₽`
14. Кликает `[В корзину]`
15. `addItem(product, { tiraj: "1000" }, 1700)` → Zustand cart store
16. Появляется toast `✓ Добавлено в корзину. Визитки «Стандарт» · 1 700 ₽ [К корзине]`
17. Иконка корзины в header пульсирует, badge `1`
18. (Этап 2 конец) Ирина продолжает искать ещё товары

### 🟡 Edge: она передумала, хочет 2 000 шт

15а. Кликает `○ 2 000 шт — 3 000 ₽`
15б. ═ API → итог `3 000 ₽`
15в. Кликает `[В корзину]` → тот же товар с новой ценой и параметрами

### 🔴 Error: БД упала при расчёте

12а. API вернул 500
12б. Цена остаётся последней успешной, под калькулятором — inline-ошибка
    «Не удалось обновить цену. Попробуйте ещё раз.»
12в. Кнопка `[В корзину]` — disabled

## 2. Flow: «Заказать баннер» (pricing_mode = calculator, per_area)

**Персона:** Алексей, маркетолог сети АЗС. Нужен баннер 3×2 м для промоакции.

### 🟢 Happy path

1. Алексей заходит сразу на `/catalog?category=naruzhnaya-reklama`
   (из органического поиска Google — SEO-страница категории)
2. Видит grid, находит «Баннер широкоформатный ПВХ»
3. Кликает → `/product/banner-pvh`
4. Видит калькулятор с полями `Ширина (м)` и `Высота (м)`
5. Вводит `3.0` в ширину
6. ═ `POST /api/products/calculate` `{ width: 3, height: null }`
7. API возвращает ошибку валидации (нужны оба поля)
8. Алексей вводит `2.0` в высоту
9. ═ API → `{ total: 5100, breakdown: [...] }`
10. Итог обновился `Итого: 5 100 ₽` со scale-анимацией
11. Раскрывает accordion «Как считается» — видит
    `3.00 × 2.00 = 6.00 м² × 850 ₽ = 5 100 ₽`
12. Кликает `[В корзину]`
13. `addItem(...)` → toast → badge

### 🟡 Edge: нестандартный размер

Алексей вводит `width = 15` (очень большой). Калькулятор обрабатывает,
но под ним появляется inline-подсказка:
> Для размеров больше 10 м мы делаем составной баннер. [Заказать расчёт →]

Клик на «Заказать расчёт» открывает `QuoteModal` с префилом параметров.

### 🔴 Error: пользователь вводит `width = -5`

Zod schema на клиенте: `width: z.number().positive()`. Под полем —
text-danger «Ширина должна быть больше 0». API не вызывается.

## 3. Flow: «Заказать стелу АЗС» (pricing_mode = quote)

**Персона:** Сергей, владелец АЗС в Сургуте. Нужна новая стела с подсветкой.

### 🟢 Happy path

1. Сергей гуглит «стела азс сургут изготовление»
2. Органика → `/product/stela-azs`
3. Видит фото готовых стел в галерее
4. Справа — «По запросу» + кнопка `[Заказать расчёт]`
5. Кликает → открывается `QuoteModal`
6. Заполняет:
   - Имя: «Сергей»
   - Телефон: «+7 912 345 67 89»
   - Email: оставляет пустым
   - Параметры: материал = «металл», комментарий «нужна с подсветкой»
7. Кликает `[Отправить заявку]`
8. ═ `POST /api/quote` с `{ product_id, name, phone, ..., comment }`
9. Запись в `calculation_requests` (status='new')
10. Notification менеджеру (Telegram/Email — backend-developer настроит)
11. Toast `✓ Заявка отправлена. Менеджер перезвонит в течение часа.`
12. Модалка закрывается
13. Analytics event `quote_request` отправлен

### 🟡 Edge: Сергей не хочет оставлять телефон

5а. Пробует отправить без телефона → валидация: «Телефон обязателен»
5б. Альтернатива: видит внизу модалки «Или позвоните сами: +7-932-424-77-40»

### 🔴 Error: API недоступен

8а. API 500
8б. Toast `❌ Не удалось отправить заявку. Позвоните: +7-932-424-77-40`
8в. Модалка остаётся открытой, данные не теряются

## 4. Flow: «Сравнить цены через фильтры»

**Персона:** Анна, офис-менеджер. Ищет самые дешёвые листовки.

### 🟢 Happy path

1. Заходит на `/catalog`
2. Открывает фильтры (desktop — sidebar, mobile — Sheet через `[⚙ Фильтры]`)
3. Ставит галку `☐ Фиксированная цена` (чтобы видеть цены сразу)
4. URL: `/catalog?pricing_mode=fixed`
5. Grid обновился, 8 товаров (SSR или client refetch)
6. Меняет сортировку на «Сначала дешёвые»
7. URL: `/catalog?pricing_mode=fixed&sort=price_asc`
8. Grid отсортирован по цене ↑
9. Листает пагинацию: `[1] [2]` → кликает `[2]`
10. URL: `/catalog?pricing_mode=fixed&sort=price_asc&page=2`
11. SSR перезагрузил страницу, скролл наверх

### 🟡 Edge: 0 результатов

Анна ставит слишком жёсткие фильтры (цена до 100 ₽).
- API возвращает 0 товаров
- Вместо grid — empty state с иконкой + «По запросу ничего не найдено»
- Кнопка `[Сбросить фильтры]` → `/catalog`

### 🟡 Edge: mobile-фильтры

На мобилке клик `[⚙ Фильтры]` открывает Sheet снизу.
Меняет поля (live-preview count в footer: `[Показать 12]`).
Кликает `[Показать 12]` → применяет + закрывает Sheet.

## 5. Flow: «Купить в 1 клик»

**Персона:** Максим, торопится. Увидел карточку визиток — хочет быстрее.

### 🟢 Happy path

1. На `/product/vizitki-standart`
2. Кликает `[Купить в 1 клик]` (вторичная кнопка)
3. Открывается `OneClickModal`
4. Заполняет Имя + Телефон (маска `+7 (___) ___-__-__`)
5. Видит подтверждение товара: «Визитки Стандарт · 1 700 ₽»
6. Кликает `[Отправить заявку]`
7. ═ `POST /api/leads` (таблица `leads`, тип `one_click`)
8. Toast `✓ Заявка отправлена, мы перезвоним в течение 15 минут!`
9. Модалка закрывается
10. Analytics: `one_click_order` event

### 🟡 Edge: параметры калькулятора не выбраны

Если Максим на `pricing_mode=calculator` и кликает «Купить в 1 клик»
без заполненного калькулятора:
- Используется **стартовая цена** из БД (`products.price`)
- В модалке вместо фикс. цены показано «от 1 700 ₽»
- В заявку пишется примечание «параметры согласовать с клиентом»
- Менеджер перезвонит, уточнит

### 🔴 Error: rate-limit hit

7а. API возвращает `429 Too Many Requests`
7б. Toast `⚠ Слишком много запросов. Попробуйте через минуту.`
7в. Модалка остаётся открытой

## 6. Flow: «Глубокий заход из поиска Google»

**Персона:** Олег, гуглит «световые буквы ханты-мансийск цена».

### 🟢 Happy path

1. Google выдача → `/product/svetovye-bukvy`
2. **Без переходов**, сразу на карточке
3. Видит breadcrumbs `Главная / Каталог / Световые буквы`
4. Видит H1, галерею фото, калькулятор периметра
5. Двигает слайдер «Периметр 300 см»
6. ═ API → «Итого: 45 000 ₽» live
7. Кликает `[В корзину]` (или `[Заказать расчёт]` если предпочитает)
8. Toast → дальше по сценарию 1 или 3

### Важно для SEO

- `/product/<slug>` должен SSR-рендериться **без JavaScript**
- JSON-LD `Product` — inline в `<head>` (Этап 2, seo-specialist сделает)
- `meta[property="og:image"]` — первое фото товара
- `meta[name="description"]` — из `products.meta_description`
- Canonical URL — абсолютный, без query-params

2. Кликает в breadcrumbs `Каталог`

через Zustand (`catalog.lastScrollY`). В Этапе 2 — не приоритет.

**Персона:** Игорь, хочет баннер нестандартного размера.

1. `/product/banner-pvh`
2. Вводит `ширина = 15 м`, `высота = 3 м`

7. Игорь добавляет «нужен с люверсами, срок до пятницы», отправляет
8. → сценарий 3

## 9. Состояния загрузки (UX ожидания)

| Действие | Длительность | Фидбек |
|---|---|---|
| Переход `/catalog` → `/product/slug` | 50–400 ms | `loading.tsx` skeleton |
| Применение фильтра | 100–800 ms | Skeleton карточек, toolbar остаётся |
| Смена сортировки | 100–500 ms | То же |
| Пагинация | 100–500 ms | То же + scroll top |
| Калькулятор (debounced) | 50–300 ms | Тихий spinner справа от «Итого» |
| Add to cart (client-side) | <16 ms | Toast |
| One-click submit | 300–1500 ms | Кнопка loading + disabled |
| Quote submit | 500–2000 ms | То же |

**Правило:** если операция дольше 300 ms — ставим visible loading state.
Если меньше 100 ms — **не показываем** ничего (мерцание хуже тишины).

## 10. Keyboard-флоу (accessibility)

### Каталог

1. `Tab` → переход по Header → CategoryChips → Filters → Toolbar → карточки → Pagination → Footer
2. На карточке — `Enter` открывает `/product/slug`
3. На чекбоксе фильтра — `Space` переключает
4. На слайдере цены — `←` `→` меняют значения (шаг 500)

### Карточка товара

1. `Tab` → Breadcrumbs → Gallery (thumbnails — стрелки) → Main action
   (`В корзину`) → Secondary (`1 клик`) → Wishlist → Tabs → Related
2. Галерея: `←` `→` переключают фото, `Enter` открывает lightbox
3. Tabs: стрелки `←` `→` между табами (Radix default)
4. Калькулятор: `Tab` по полям, `Enter` не сабмитит (форма не submit'ится,
   цена считается live)

### Модалки

- Открытие: Radix focus на первом поле автоматически
- `Tab` — по полям модалки (focus-trap)
- `Shift+Tab` — назад
- `Esc` — закрыть (Radix default)
- `Enter` в последнем поле — submit (если все валидны)

| Event | Когда |
